📖 O Modelo Cliente/Servidor & Protocolos

A Internet baseia-se num fluxo simples: o cliente (Navegador/Front-End) realiza uma Requisição (Request) e o servidor (Back-End) retorna uma Resposta (Response) com arquivos ou dados (HTML/JSON).

Os protocolos garantem o formato unificado dessa comunicação. O HTTP é a base na web (frequentemente com sua versão mais escalável e segura HTTP/3 utilizando QUIC). Os Métodos HTTP principais são: GET (buscar dados e recarregar páginas), POST (enviar dados novos), PUT/PATCH (atualizar) e DELETE.

Por trás das URLs amigáveis (ex: https://www.meusite.com.br) existe o sistema DNS (Domain Name System) que converte este texto em um endereço IP, utilizando hierarquia de TLD (Top Level Domain, ex .br) e SLD (.com).

// Request Example GET /articles/hello.html HTTP/1.1 Host: www.example.com

💼 Carreiras em Desenvolvimento WEB

A divisão entre cliente e servidor reflete perfeitamente as profissões no mercado e na arquitetura web:

  • Front-end Developer: Cria a interface visual e o rosto da aplicação. Interagindo com navegadores e APIs. É o "Dev do Cliente". Usa HTML, CSS e React/Angular.
  • Back-end Developer: Desenvolve a lógica e integra com Banco de Dados e segurança. Processando as Requisições. Usa Node.js, Java, Python ou .NET.
  • Full Stack Developer: Atua ponta a ponta no Front e no Back-end entregando soluções completas.
  • QA / DevOps / Arquiteto: Responsáveis pela infraestrutura virtual, testes automatizados e topologia arquitetural moderna e escalável.

Hands-on: Analisando o DevTools Numérico Real

Utilizando a ferramenta integrada dos navegadores (Chrome, Firefox, Safari) é essencial debugar e analisar tráfego de requisições. Cumpra os passos e envie ao professor:

  1. Abra o navegador e ative o painel do DevTools (F12).
  2. Acesse um site contendo informações de request (ex: baitabusca.com) e observe a aba Network começar a preencher.
  3. Filtre os tipos por Doc (Document) para encontrar a primeira requisição que entrega a página HTML. Analise também suas abas auxiliares como Headers para encontrar a URL base, os métodos verbos HTTP enviados e os códigos Status da resposta do servidor (ex: status 200 significando sucesso pleno em HTTP Code).
  4. Com a aba Network focada no contexto, exporte o seu registro em um arquivo .HAR que armazena essas logs na sua máquina.
  5. Tire 3 prints de evidência: a lista de requests da aba Network inteira, a janela de Headers lateral e a janela de Response com o retorno cru.
  6. Faça uma requisição extra usando Formulário com botão ou acesso via POST para analisar o corpo da comunicação.
  7. Zipe todos os prints com o arquivo .HAR para e-mail e envie ao professor: [seu nome] - [turma] - Usando DevTools.

📖 Integração Contínua (CI/CD) e Controle de Fluxo

O Git é o standard de mercado para sistema de controle de versão livre e open source (VCS) distribuído de alta perfomance. Todo desenvolvedor rastreia seu desenvolvimento via commits diários, seja no terminal ou ferramentas visuais, trabalhando num método colaborativo (Trunk-Based Development) sem conflitar arquivos alheios quebrando o site.

Quando configurado com repositórios e branches numa base contínua online de integração e entrega (Pipelines DevSecOps CI/CD), seu novo código injetado é enviado ao ar (Cloud, VPS) em minutos sem necessidade de transferências manuais de arquivos via FTP. O repositório enxerga os comandos de git push e inicia scripts de Pipeline.

🚀 Pipeline de Deploy Básico

$ git add . $ git commit -m "feat: adicionando novas rotas" $ git push origin main # Servidores render.com capturam o push e atualizam o deploy!

📖 O Modelo de Caixas (Box Model) e Regras

Nenhuma tag foge dessa matemática do CSS Box Model. Toda as larguras/alturas são divididas em: Content (o recheio real), circulado pelo Padding (o espaçamento interno que engorda a tag), delimitado pelas linhas do Border e finalmente o Margin (o espaço estéril invisível externo que empurra os vizinhos da tag de longe).

Sempre garanta que seus Seletores atinjam alvos de forma hierárquica usando Classes (.minha-classe) e ID (#meu-id). Cuidado ao setar dimensões rígidas: o uso de Medidas Relativas globais % e de tela vw e vh (Viewport Width/Height), rem (Relativo ao root), mais as operações vivas com calc() são vitalícios e mandatórios frente ao pixel-perfect.

Hands-on: Montando Layout 2D Responsivo com CSS Grid

Implemente um novo projeto limpo chamado layout-grid.html usando o layout bidimensional mais avançado do mercado, o CSS Grid:

  1. Na raiz zere o espaço global colocando a formatação body HTML margin: 10% no stylesheet inline ou target header local.
  2. Estruture a dom criando uma caixa parente principal flexivel: <div class="wrapper"> e a converta para display do tipo container atômico no estilo com display: grid;.
  3. Especifique as tracks de espaço fracionadas dela com propriedades para layout: grid-template-columns: 10% 10% 10% 10% 10%; (Distribuindo uniformemente em blocos fluidos).
  4. Adicione um respiro padronizado (calhas) entre cada item via declaração no wrapper class css: grid-gap: 10vw; e pinte o fundo background-color em tons dark gray.
  5. Construa na DOM cerca de 5 filhos blocos internos sequencias <div class="box a">A</div> repetidos sequencialmente. Pinte os quadrados classes com fundo branco, arredonde pontas via border-radius: 5px preveja espaçamento padding: 10% interno dentro do bloco para a tipografia "A" saltar longe da borda interna.
  6. Realize a visualização técnica expandindo e encolhendo toda janela desktop ou testando via device mode (mobile emulate DevTools F12) observando flexibilidade do esqueleto global CSS nativo.

📖 Variáveis e Coleções (Arrays)

O JavaScript possuí forte tipagem não-declarada. Historicamente variáveis eram salvas de forma confusa no escopo Root Memory via palavra var sofrendo manipulações sujas pelo Hoisting (içamento orgânico do interpretador que causava vazamento do dado global). Hoje usa-se apenas blocos protegidos criando constantes matemáticas com const PI = 3.14 e variáveis dinâmicas seguras com let idade = 20 que morrem ao final do bloco de execução.

No domínio das manipulações massivas de dados, o JS detém as Listas ou Vetores baseados em JSON, ou seja Arrays. Algumas das funções utilitárias embutidas no protótipo das arrays que usaremos frequentemente incluem: .forEach() (passar para ler todos os itens), .map() (modificar os dados e retornar nova lista editada), .filter() (descarta falsos deixando só o procurado) e .reduce() (acumula todos os itens de uma lista em uma só conta matemática/resultado).

Hands-on: Funções Utilitárias e Dates

Com base nas explicações da aula, o aluno deverá programar (em seu própio arquivo script.js vinculado ao HTML) e executar as cinco funções abaixo. Passando os parâmetros algorítmicos corretos simulando regras do mundo real (Strings, Ints, Datas). Pratique a checagem no console.log() do DevTools ou na UI abaixo:

  1. nomeCompleto(nome, sobrenome): Recebe strings isoladas. Deve gerar concatenação string formatada com as crases para interpolação nativa (`${nome}...`).
  2. calcularIdade(ano): Recebe int. Subtraí o ano nascido pelo tempo corrente de execução do motor para deduzir a idade estática da pessoa sem analisar aniversário.
  3. jaFezAniversario(mes, dia): Verificação relacional condicional usando If/Else encadeado ou de blocos. Invoque o objeto de relógio new Date() obtendo dia corrente do PC. Retorna flag true se a pessoa local for mais velha no ano em questão.
  4. diasParaOAniversario(mes, dia): Trabalhando o JS Date em matemática. Transforma anos em métrica universal usando .getTime() (milissegundos limpos). Calcula as distâncias baseadas e divide sua somatória (1000 * 60 * 60 * 24). Retornando a soma total numérica final de dias isolados.
  5. mensagem(): Função pai (wrapper logic) instanciando/invocando suas co-vizinhas. Monta a string "O seu nome completo é X" atrelado à "Sua idade atual é Y e seu níver...", avaliando condicional do JaFezAniversario() nativo. Retornando os anos que falta pra atingir os cem de idade.
