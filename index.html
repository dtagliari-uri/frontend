História da Internet e Evolução da Web

Da rede militar ARPANET ao HTTP/3 moderno — uma linha do tempo interativa pelos marcos que moldaram a internet como conhecemos hoje.

🏛️
1969

ARPANET

A primeira mensagem "LO" foi enviada entre UCLA e Stanford — o início da rede que se tornaria a internet. Criada pelo Departamento de Defesa dos EUA para descentralizar comunicações.

📡
1983

TCP/IP — O Dia da Bandeira

A ARPANET adotou oficialmente o protocolo TCP/IP em 1 de janeiro de 1983 ("Flag Day"). Este conjunto de protocolos tornou-se a base universal da comunicação em rede.

🌐
1991

World Wide Web — Tim Berners-Lee

Pesquisador do CERN, Tim Berners-Lee propôs e implementou o HTTP e o HTML, criando o World Wide Web. O primeiro site foi info.cern.ch, ainda online hoje.

🖼️
1993

Mosaic Browser

O Mosaic foi o primeiro navegador gráfico a exibir imagens inline com texto, tornando a web acessível ao público em geral. Desenvolvido no NCSA pela equipe de Marc Andreessen.

🚀
1994–1995

Netscape Navigator & Guerra dos Browsers

Netscape dominou o mercado com 90% de share. A Microsoft respondeu com o Internet Explorer 1.0 em 1995, iniciando a "Browser Wars" que acelerou dramaticamente a inovação na web.

⚡
2008–2015

Chrome, Firefox, Safari & Edge

Google Chrome (2008) revolucionou com o motor V8 e atualizações silenciosas. Firefox (2004) trouxe extensões e código aberto. Safari e Edge completam o ecossistema moderno de browsers.

🔮
Hoje

HTTP/3, WebAssembly & Web Platform

HTTP/3 usa QUIC sobre UDP para conexões ultrarrápidas. WebAssembly traz código nativo ao browser. A Web é agora a plataforma de desenvolvimento de aplicativos mais universal da história.

Browsers, Protocolos e Modelo Cliente-Servidor

Como HTTP, HTTPS, DNS e o modelo cliente-servidor formam a espinha dorsal de toda comunicação na web moderna.

Simulação: Fluxo de Requisição HTTP

💻
Browser (Cliente)
──── GET /index.html ────▶
◀──── 200 OK + HTML ────
🖥️
Servidor (Back-end)
HTTP/1.0
HTTP 1.0

Uma conexão TCP por requisição. Sem cabeçalhos de persistência. Muito ineficiente para páginas complexas (1991–1996).

HTTP/1.1
HTTP 1.1

Conexões persistentes (keep-alive), pipelining e compressão gzip. Padrão por mais de 10 anos (1997–2015).

HTTP/2
HTTP 2

Multiplexação de streams, compressão de headers HPACK, server push. 2× mais rápido que HTTP/1.1 (2015+).

HTTP/3
HTTP 3 + QUIC

Baseado em UDP via QUIC. Zero Round-Trip Time na reconexão. Ideal para mobile e redes instáveis (2022+).

HTTPS
HTTPS + TLS

HTTP com camada TLS para criptografia end-to-end. O cadeado verde no browser. Obrigatório em produção.

DNS
DNS

Domain Name System — converte www.google.com em 142.250.1.1. A "agenda telefônica" da internet.

Sistemas de Layout CSS

Flexbox Demo

📍

static

Posição padrão — o elemento segue o fluxo normal do documento.

↔️

relative

Posicionado em relação à sua posição original. Não sai do fluxo.

🎯

absolute

Remove do fluxo. Posicionado em relação ao ancestral relative.

📌

fixed

Fixo na viewport — não muda com o scroll. Ideal para navbars.

🔒

sticky

Misto entre relative e fixed — "gruda" ao atingir o threshold de scroll.

🔢

z-index

Controla a ordem de empilhamento (stacking). Maior valor = na frente.

Proteja sua aplicação e os dados dos usuários contra os ataques mais comuns da web.

Criptografa a comunicação entre cliente e servidor, impedindo interceptações (Man-in-the-Middle).

O Pensamento Reativo (Exemplo React)

function Counter() { const [count, setCount] = useState(0); return ( <button onClick={() => setCount(count + 1)}> Contagem: {count} </button> ); }

Diferente do Vanilla JS, onde você manipula o DOM manualmente (appendChild), em frameworks você apenas atualiza o Estado e a UI reage automaticamente.
