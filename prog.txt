World Wide Web nasceu no início dos anos 1990, criada por Tim Berners-Lee no CERN (Organização Europeia para a Pesquisa Nuclear), com a proposta de facilitar o compartilhamento de informações entre cientistas. Uma URL, ou Localizador Uniforme de Recursos, é um endereço que identifica um recurso na internet, como uma página web, imagem ou arquivo. É basicamente o "endereço" que você digita no navegador para acessar um site ou recurso específico
Estrutura da URL:
Protocolo: Indica o método usado para acessar o recurso (por exemplo, http, https, ftp);
Domínio: É o nome do servidor onde o recurso está hospedado (por exemplo, example.com);
Caminho: Especifica a localização do recurso no servidor (por exemplo, /articles/my-article.html);
String de Consulta (opcional): Fornece parâmetros adicionais para a solicitação (por exemplo, ?id=1234&category=shoes&color=blue);
Âncora (opcional): permite que você se conecte a uma parte específica de uma página da web ou de um documento (por exemplo, #material-theming).

Um protocolo é um conjunto de regras e procedimentos que governam a troca de dados entre diferentes dispositivos e sistemas, permitindo que eles se comuniquem e interajam de forma padronizada. É como uma linguagem comum que computadores e outros dispositivos usam para trocar informações na internet e em redes locais.

O HTTPS é a versão segura do HTTP. Ele mantém a mesma função, trocar informações entre cliente e servidor, mas adiciona uma camada de segurança usando o TLS (Transport Layer Security). O “cadeado” no navegador indica que a conexão é segura e que o site possui um certificado digital válido.
Para ter HTTPS com TLS (Transport Layer Security) em seu site, você precisa obter e instalar um certificado SSL/TLS. Isso envolve adquirir o certificado de uma Autoridade Certificadora (CA), configurar seu servidor web para usá-lo e garantir que seu site esteja acessível através do protocolo HTTPS
HTTPS garante confidencialidade (dados não podem ser lidos por terceiros), integridade (dados não são alterados) e autenticação (confirma a identidade do servidor).

Em sua essência, o DNS resolve um problema simples, mas crucial: humanos preferem nomes memoráveis, enquanto computadores precisam de números. Em 2004, alguns desenvolvedores da Apple, Mozilla e Opera formaram o WHATWG (Web Hypertext Application Technology Group) para criar o HTML5 e montaram a comunidade de manutenção e desenvolvimento de padrões web, incluindo DOM, Fetch e HTML.

A versão do HTML 5 virou um living standard (padrão vivo), o que significa que está sempre evoluindo, dentro da mesma versão HTML5.

As tags pareadas consistem em duas instruções:
Uma tag de abertura (também chamada de tag inicial) que marca o início de um bloco;
Uma tag de fechamento que é igual à tag de abertura, mas com uma barra adicional /
<html>, <body>, <p>, <div>, <a>, <h1>

Tags não pareadas, também conhecidas como tags fechadas, não têm conteúdo interno;
Elas formam elementos gráficos ou símbolos em uma página. Assim, as tags HTML não pareadas possuem apenas uma tag de abertura.
<img>, <br>, <hr>, <input>, <meta>
O HTML não sofre de erros de sintaxe porque os navegadores o analisam permissivamente, o que significa que a página ainda é exibida mesmo se houver erros de sintaxe.

O que é DevOps e DevSecOps?
DevOps é a união de desenvolvimento (Dev) e operações (Ops) através de uma cultura, práticas e ferramentas que automatizam e integram os processos de desenvolvimento de software e operações de TI.

DevSecOps é a integração proativa da segurança (Sec) em todas as fases do ciclo de vida de desenvolvimento de software (DevOps).
Foco:
DevOps prioriza velocidade e eficiência. 

DevSecOps prioriza velocidade com segurança proativa.
Segurança: 
No DevOps, a segurança muitas vezes é uma etapa final (reativa). 

No DevSecOps, é contínua e automatizada (proativa).
Responsabilidade: 
DevSecOps transforma a segurança em uma responsabilidade de todos, não apenas da equipe de segurança

Por que usar CI/CD ?
(Integração Contínua e Entrega/Implantação Contínuas)
Redução drástica de erros humanos: A automação de testes e deploys elimina falhas manuais comuns e garante que o código em produção siga padrões rigorosos.

Time-to-Market acelerado: Reduz o ciclo de entrega de semanas ou meses para apenas minutos

Detecção precoce de bugs: Problemas são identificados e corrigidos assim que surgem, antes que se tornem complexos e caros de resolver.

Feedback contínuo: Permite que desenvolvedores vejam o progresso em tempo real e recebam feedback rápido dos usuários finais por meio de lançamentos frequentes.

Melhoria na colaboração: Facilita o trabalho de múltiplos desenvolvedores no mesmo projeto, resolvendo conflitos de integração automaticamente e aumentando a produtividade da equipe.

Segurança integrada (DevSecOps): Pipelines modernos permitem incluir verificações de segurança automáticas em cada etapa, mitigando vulnerabilidades antes que o software seja publicado.

Quando usar wireframes
As equipes de UX costumam usar wireframes nos estágios iniciais de design. Depois que um wireframe se torna um mockup, não há motivo para voltar atrás — a menos que você esteja redesenhando o produto!

Quando usar mockups
Designers de UX criarão mockups a partir de wireframes em preparação para a prototipagem de alta fidelidade. As equipes de UX também usam mockups para apresentar designs a stakeholders ou clientes.

Quando usar protótipos
Você encontrará protótipos ao longo do processo de design, da concepção à entrega final. Sem protótipos, os testes de usabilidade não existiriam. As equipes de UX criam protótipos de baixa e alta fidelidade em todas as etapas para testar, obter feedback, fazer alterações e repetir.

Protótipos de baixa fidelidade raramente saem do departamento de UX, enquanto as equipes usam regularmente protótipos de alta fidelidade para testes ou apresentações para clientes e stakeholders.

Valores e Unidades
CSS3: Estilização, Layout e Responsividade
É crucial escolher a unidade correta para cada situação, utilizando unidades relativas para designs responsivos que se adaptam a diferentes tamanhos de ecrã. 
Unidades Absolutas: Medidas fixas, independentes de outros elementos da página: 
px: (pixels): A unidade mais comum para telas, representando um ponto na tela. 
cm: (centímetros), mm (milímetros), in (polegadas), pc (picas), pt (pontos): Usadas principalmente para impressão, garantindo que uma medida seja exatamente o que se pretende, sem depender do dispositivo. 
Unidades Relativas: São medidas que variam em relação a um valor de referência. 
%: (porcentagem): A medida é uma fração da dimensão do elemento pai, sendo útil para definir larguras e alturas relativas. 
vw: (viewport width): 1% da largura da viewport (a área visível do navegador). 
vh: (viewport height): 1% da altura da viewport. 
vmin e vmax: 1% da menor (vmin) ou maior (vmax) dimensão da viewport. 
em: Refere-se ao tamanho da fonte do elemento pai acima dele. O 1em é igual ao font-size desse elemento. 
rem: É uma unidade de medida relativa que define o tamanho de um elemento com base no tamanho da fonte do elemento raiz (root) da página

Uma pseudo-classe CSS é uma palavra-chave adicionada a seletores que especifica um estado especial do elemento.

.lista li:nth-child(2) { background: yellow; }    /* segundo item */
.lista li:nth-child(odd) { background: #ddd; }   /* itens ímpares  */
.lista li:nth-child(even) { background: #bbb; }   /* itens pares */

input:checked + label { color: green; }   /* label ao lado de checkbox */
input:disabled { background: #ccc; }   /* input desabilitado */
p:not(.paragrafo) { color: gray; }   /* todos que NÃO têm classe */

a:link { color: blue; }   /* link ainda não visitado */
a:visited { color: purple; }   /* link já visitado */
a:hover { color: orange; }   /* quando passa o mouse no link */

Tudo em CSS tem uma caixa ao redor, as 4 principais partes do modelo de caixa são:

Conteúdo (Content): É a área central da caixa, onde o texto, imagens ou outros elementos visíveis são exibidos. As propriedades width e height controlam as dimensões desta área.

Preenchimento (Padding): O espaço que fica entre o conteúdo e a borda da caixa, criando um afastamento interno. É controlado pela propriedade padding.

Borda (Border): Uma linha que circunda a área de conteúdo e o preenchimento, delimitando a caixa. Controlada pelas propriedades border.

Margem (Margin): A área mais externa da caixa, que define o espaço entre a caixa do elemento e os outros elementos na página. É controlada pela propriedade margin.

Introduzido no ECMAScript 6 (ES6), o let é uma alternativa moderna ao var e resolve alguns problemas;
Possui escopo de bloco, ou seja, só estão acessíveis dentro do bloco em que foram definidas.

Você pode reatribuir uma variável declarada com let, mas não pode redefini-la no mesmo escopo.

function jaFezAniversario(dia, mes) {
 const hoje = new Date();
 const diaAtual = hoje.getDate();
 const mesAtual = hoje.getMonth() + 1; // meses começam em 0

 if (mesAtual > mes) {
   return true;
 } else if (mesAtual === mes && diaAtual >= dia) {
   return true;
 } else {
   return false;
 }
}

// teste
console.log(jaFezAniversario(10, 3));

✅ 4. Dias para o aniversário
function diasParaAniversario(dia, mes) {
 const hoje = new Date();
 const anoAtual = hoje.getFullYear();

 let proximoAniversario = new Date(anoAtual, mes - 1, dia);

 // se já passou, vai para o próximo ano
 if (proximoAniversario < hoje) {
   proximoAniversario = new Date(anoAtual + 1, mes - 1, dia);
 }

 const diff = proximoAniversario - hoje;
 const dias = Math.ceil(diff / (1000 * 60 * 60 * 24));

 return dias;
}

// teste

👉 Regex é um padrão de busca em texto Usado para:
validar dados (email, senha, CPF)
encontrar palavras
substituir partes de texto

🧩 Estrutura básica
/padrao/

Exemplos:
/a/ → encontra a letra "a"
/abc/ → encontra "abc"
/[0-9]/ → números
/[a-z]/ → letras minúsculas

⚙️ Métodos principais
.test()     // retorna true ou false
.match()    // retorna o que encontrou
.replace()  // substitui texto
Expressões Regulares em Javascript
1. Verificar se tem número ( Esperado: true)
let texto = "Produto 123";
let regex = /\d/;
regex.test(texto);

2. Encontrar todas as letras (Esperado: ["A", "B", "C"])
let texto = "ABC123";
texto.match(/[A-Z]/g);

3. Validar email simples (Esperado: true)
let email = "teste@email.com";
let regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
regex.test(email);

4. Substituir números (Resultado: "Preço: XXX reais" )
let texto = "Preço: 123 reais";
texto.replace(/\d+/g, "XXX");

class Aluno extends Pessoa {
  constructor(nome, idade, curso) {
    super(nome, idade); // chama a classe pai
    this.curso = curso;
  }

  estudar() {
    return `${this.nome} está estudando ${this.curso}`;
  }
}

let a1 = new Aluno("Ana", 22, "TI");
console.log(a1.estudar());

try - bloco onde colocamos código que pode gerar erro.
catch - captura o erro e permite tratá-lo sem quebrar a aplicação.
throw - lança um erro (nativo ou personalizado).
finally - executa sempre, com ou sem erro.
try {
   let x = y + 1; // y não existe → ReferenceError
} catch (err) {
   console.log("Ocorreu um erro:", err.message);
} finally {
    console.log("Sempre executa, com ou sem erro.");
}

Tratamento de Erros Try/Catch
JavaScript: Sintaxe Básica e Manipulação do DOM
	•	try protege o código.
	•	catch lida com erros.
	•	throw cria seus próprios erros.
	•	finally roda sempre, garantindo limpeza e previsibilidade.
